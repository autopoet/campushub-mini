# CampusHub Mini: Social Hall Design Spec
## 校园大厅设计指南 (V2.0 - 社交降噪版)

### 1. 核心视觉哲学 (Aesthetics)
- **风格定义**：便利贴 Post-it / 潮流大字报 / 极简方块。
- **排版原则**：
  - **去 Emoji 化**：使用 `[ STUDY ]` 等纯文字标签替代图标，提升 Premium 感。
  - **大字导向**：关键信息一秒扫完（例如：已有一名后端，急缺：Vue前端）。
  - **黄金律动**：全页面采用 8rpx 倍率系统。表单项间距统一为 `48rpx`，输入框内边距 `30rpx/40rpx`，创造极致呼吸感。
  - **动态背景**：广场卡片采用随机柔和色彩（淡紫、淡蓝、淡黄），自带微旋转动效。

### 2. 社交准入机制 (Registration Strategy)
- **渐进式补全 (Progressive Disclosure)**：
  - **注册期 (Low Friction)**：仅要求头像、昵称、学校、年级。**联系方式设为选填**，解决首跳隐私焦虑。
  - **动作期 (On-Demand)**：当用户执行“发布动态”或“戳一戳对手”时，系统强制检查联系方式。若为空，则引导补全。
- **联系方式 (Contact UI)**：
  - **分类式交互**：采用 [手机] [QQ] [微信] 标签切换。
  - **优先级排序**：手机号 > QQ > 微信。
  - **多态存储**：支持同时保存三种方式，已填写的分类标签右上角亮起“青色小圆点”。

### 3. 名词术语规范 (Terminology)
- **通用称呼**：全线去除“队友”、“组队”，统一改为 **“搭子”**（如：开启搭子之旅、寻找搭子）。
- **交互动作**：点击动作定义为 **“戳他 (Poke)”**。

### 4. 交互模型 (Social Loop)
- **名片投递逻辑**：
  1. 用户在广场看到心仪帖，点击“戳他”。
  2. 此动作即为“投递个人名片”。
  3. 被戳者在“信号中心”查看投递者资料。
  4. 若被戳者点击“回戳/同意”，双方自动交换联系方式并一键复制到剪贴板。

### 5. 边界处理 (Edge Cases)
- **环境隔离**：首页注入 `boot-loading` 逻辑，确保云服务初始化前不黑屏。
- **数据快照**：发帖时对发布者当时的学校、头像进行“拍照快照”，确保帖子数据的稳定展现。
- **校友背书**：所有便利贴底部必须自动带上 `@ 发布者学校`，增强信任感。
